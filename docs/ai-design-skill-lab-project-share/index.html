WHY先讲问题

AI 设计最大的痛点，不是不会生成

真正麻烦的是：生成之后，很难说明来源、判断、迭代和下次如何复用。

结果散落图片、Prompt、截图、聊天记录分散在不同地方，后续很难找。
判断不可见为什么选这个方向、为什么拒绝另一个方向，经常没有记录。
经验不回流一次项目做完，下一个项目还是从零开始问、找、试。
这个项目要解决的，是让 AI 设计工作可复盘、可验证、可复用。
从“生成结果”升级到“沉淀系统”02 / 12
POSITION项目定位

AI Design Skill Lab 是一个设计生产实验室

它把设计经验拆成可以被人和 Agent 共同读取、执行、验证和版本化的工程资产。

Design Data Factory负责资料、Pattern、Rule、Prompt、Run 记录等资产沉淀。
Harness Runtime负责模拟一次 AI 设计任务如何计划、生成、评价、复审、归档。
Output Skill负责把判断变成可发布、可演示、可归档的输出包。
核心不是“多做一点内容”，而是“让一次工作成为资产”。03 / 12
STRUCTURE四层结构

从资料到输出，中间分成四层

先接住输入，再沉淀资产，再执行生产流程，最后形成可交付输出。

资料层

接住未消化输入：文章、brief、截图、外部项目和历史导入。

00_Inbox_Staging / imports / references

资产层

把经验结构化：可复用模式、规则、提示词、运行记录和 schema。

references / shared

生产层

让流程可执行：CLI 处理资产，Runtime 模拟任务生命周期。

scripts / harness

输出层

形成可展示成果：图文包、HTML deck、评估文档和发布文案。

docs / demo / tests
理解这四层，就能理解目录为什么这样拆。04 / 12
PIPELINE资产生产链路

Pipeline 负责把资料加工成资产

它是一组单步 CLI，每一步都可以独立运行、检查和复用。

scan盘点原始资料，生成 manifest 和 Asset 记录。
run结合 brief、Prompt、Pattern，生成设计 Plan。
critic用规则和 rubric 评价方向是否值得继续。
archive把通过评价的方向归档为 Pattern。
generate把方向继续转成 Visual 或输出素材。
入口主要在 scripts/，共用能力在 shared/。05 / 12
HARNESS任务生命周期

Harness 负责验证“任务怎么跑完”

它更像一个运行时外壳，用状态机描述一次 AI 设计任务从目标到归档的全过程。

Goal明确这次任务要解决什么问题。
Planner拆解任务，给出执行计划。
Generator生成候选设计或 mock 结果。
Critic评价结果，失败时触发 retry。
Review需要人工边界时停下，最后 Archive。
当前是 deterministic mock lifecycle，不调用 Pipeline CLI。06 / 12
BOUNDARY当前边界

现在先分开跑，未来再打通

这是当前仓库最重要的架构边界：不要误以为 Harness 已经接入了 Pipeline。

两者当前互不依赖

先把边界讲清楚，项目协作才不会混乱。07 / 12
OUTPUT SKILL输出规范

Output Skill 负责把判断变成交付物

它定义小红书图文、HTML 导出、PPT / Presentation Output 应该如何组织文件、记录来源和验证质量。

小红书图文HTML/CSS 源文件，浏览器截图导出 1080 x 1350 PNG。
HTML Deck横向 presentation，可浏览、可投屏、可导出封面和关键页。
输出检查检查结构、尺寸、中文可读性、来源记录和复用价值。
输出不是最后一步截图，而是可回到系统里的资产包。
输出任务先读 docs/OUTPUT_SKILL_SPEC.md。08 / 12
END TO END一次任务流动

从 brief 到复用，是一条闭环

这个仓库关心的不是“这次做完”，而是“这次之后，下次能不能更快更稳”。

brief目标、资料、限制、场景进入 staging。
reference隔离外部资料，判断是否值得吸收。
plan生成方向，并记录消耗了哪些资产。
render转成 HTML、图文、deck 或 visual。
reuse通过评价的经验沉淀为 Pattern / Rule / 文档。
brief -> reference -> analysis -> plan -> prompt -> layout -> render -> review -> archive -> reuse09 / 12
MAP目录怎么读

MAP.md 是第一事实源。其它目录围绕资料、资产、生产、输出分工。

给新人一句话：先看 MAP，再按任务进目录。10 / 12
VALUE分享价值

它对团队真正有价值的地方

不是某个模型调用，而是围绕模型调用建立了一套可协作、可审计、可复用的工作方式。

设计资产化把经验从聊天记录和文件夹里提取出来，变成可检索资产。
协作可追踪Git、README、run notes、测试和导出检查让工作可复查。
AI 可治理让 Agent 不只是生成，也能按规则执行、验证和归档。
这就是 DesignOps 与 AI Runtime 结合的意义。11 / 12
