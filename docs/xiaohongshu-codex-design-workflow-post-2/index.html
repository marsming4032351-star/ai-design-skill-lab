TWO ENTRANCES

Codex 可以理解成
两种入口

⌘
终端版 Codex 更像会操作项目文件的执行助理。适合生成文档、整理资料、导出图文、沉淀工作流。
↗
桌面端 / ChatGPT 更像低门槛对话入口。适合先讲需求、改文案、拆流程、理解下一步怎么做。
不是二选一，是分工不同02 / 06
HOW TO CHOOSE

小白先这样选
就不会卡住

聊

想清楚需求

先用桌面端 / ChatGPT：问问题、改文案、拆选题、整理 5 张卡片结构。

做

真的生成文件

用终端版 Codex：读项目、写 HTML、导出 PNG、整理到 docs 或 Obsidian。

合

最顺的方式

桌面端先把需求讲明白，终端版再把它做成可复用交付。

桌面端负责想清楚，终端版负责做出来03 / 06
MINDSET SHIFT

别先学终端
先学会下任务

01我有什么资料？微信文章 / PPT / 案例 / 灵感
02我要什么结果？小红书图文 / 提案结构 / 品牌分析
03谁来看？小白用户 / 客户 / 设计师 / 老板
04先输出什么？主题、卡片结构、文案，再做图片
需求说清楚，比会敲命令更重要04 / 06
COPY THIS

新手可以直接
复制这段话

我有一篇微信文章，想做成小红书图文。
请先帮我提炼主题、目标用户、5 张卡片结构和发布文案。
不要直接生成图片，先给我一个适合小白理解的内容方案。
这段话适合先发给桌面端 / ChatGPT。等结构确认后，再让终端版 Codex 做成 HTML 和 PNG。
./scripts/capture.sh wechat "文章标题" "文章内容或链接"
