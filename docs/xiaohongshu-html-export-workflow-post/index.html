DESIGN DATA FACTORY 小红书图文工作流 03

别再让 AI 直接生成满屏中文

更稳定的方法是：HTML/CSS 排版，再用浏览器截图导出 1080 x 1350 的小红书图片。

HTML
负责文字和版式 标题、分段、编号、检查项都可改
PNG
负责发布和收藏 统一尺寸、稳定清晰、方便复用

适合：教程图文、工具拆解、策划流程、品牌方法论。

WHY 先讲判断

纯生图的问题，不是不好看

问题是它很难稳定生成可读中文，也很难反复修改。小红书教程图文最重要的是：读得清、改得动、能复查。

纯生图适合

氛围、主视觉、封面背景、产品质感、情绪画面。

HTML 适合

长标题、中文正文、步骤清单、命令代码、对比表。

结论：主视觉可以生图，正文和结构交给 HTML。

STRUCTURE 一张图就是一个 section

HTML 里先把每张卡片固定成 1080 x 1350

不要先写一整页文章。把每张图当成一个可截图的模块，设置固定尺寸和独立 id。

<section class="card" id="card-01"> <h1>封面标题</h1> <p>这张图只讲一个观点</p> </section> .card { width: 1080px; height: 1350px; }

核心原则：一张卡只讲一个任务，不让文字挤在一起。

EXPORT 浏览器截图导出

用 Playwright 按 id 截图

先打开 HTML，再分别截取每个卡片节点。这样导出的 PNG 尺寸稳定，排版也和浏览器看到的一致。

python3 -m http.server 8765 playwright screenshot "#card-01" \ --filename "cards/card-01.png" file cards/card-01.png

如果你用 Codex，可以让它自动完成打开、截图、检查尺寸。

CHECKLIST 发布前最后一遍

导出后别急着发，先查 6 件事

尺寸必须是 1080 x 1350，比例 4:5。
中文标题、正文、代码都要清晰可读。
溢出检查文字有没有压线、遮挡或出框。
一致卡片内容要和发布文案互相对应。
来源外部素材要写清用途，不混进主项目。
复用README 写清怎么改、怎么导出。

这就是我现在做小红书图文最稳定的一条链路：brief -> HTML -> browser -> PNG -> archive。
