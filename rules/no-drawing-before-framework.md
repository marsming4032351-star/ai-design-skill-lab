# Rule：No Drawing Before Framework

本规则适用于 Real-World Driven Design Harness。它规定：在完成 research、interpretation、framework 三个阶段之前，不允许进入视觉生成阶段。

## 核心规定

AI 不允许一上来画图。任何 Visual Agent、效果图生成、风格探索、图像提示词、HTML deck 视觉稿或空间渲染，都必须等到以下三个阶段完成后才能开始：

1. `10_research/`：完成研究阶段。
2. `20_interpretation/`：完成解释与设计机会阶段。
3. `30_framework/`：完成设计框架阶段。

如果缺少上述任一阶段，任务必须返回前置阶段，而不是继续生成图像。

## 可以进入 Visual Agent 的条件

只有同时满足以下条件，才允许进入 Visual Agent：

- 已有 Human Sensor 输入，包含现场、甲方、人群或运营约束。
- Research Agent 已输出事实、背景、来源和资料缺口。
- Interpretation Agent 已把资料转化为矛盾、情绪、动机和设计机会。
- Framework Agent 已输出设计定位、核心命题、关键词、空间策略、视觉策略和运营约束。
- Framework 中明确写出“不做什么”。
- Visual Agent 的任务说明能引用 Framework 中的具体判断。
- 项目负责人或执行 Agent 明确记录进入视觉阶段的理由。

## 不允许进入 Visual Agent 的情况

出现以下任一情况，应拒绝进入视觉生成：

- 只有甲方一句需求，没有现场和人群资料。
- 只有参考图，没有研究和解释。
- 只有“高级、年轻、科技感、自然、松弛”等风格词。
- 没有明确设计机会。
- 没有运营、预算、维护或落地约束。
- 无法说明视觉方向回应了哪个现实问题。
- 只是为了快速产出一张好看的图。

## 美颜化设计的风险

“美颜化设计”指方案主要通过表面风格让结果显得高级、精致或流行，但没有解决真实问题。

常见风险：

- 把场地问题变成氛围滤镜。
- 把地方历史变成装饰符号。
- 把用户需求变成抽象情绪词。
- 把甲方目标变成漂亮提案话术。
- 把运营约束藏起来，只展示理想状态。
- 把空间尺度、施工、维护、翻台、设备和预算排除在画面之外。

## 美颜化设计的识别标准

如果一个方案符合以下任意 3 条，应标记为高风险：

- 看起来可以放在任何项目里。
- 无法解释为什么只能发生在这个场地。
- 没有回应真实人群行为。
- 没有回应甲方商业或组织目标。
- 没有说明施工、维护或运营条件。
- 只使用流行风格词描述方案。
- 效果图很好看，但研究、解释和框架找不到对应关系。
- Critic Agent 无法追溯设计选择的现实依据。

## 处理方式

当发现美颜化风险时：

1. 暂停视觉生成。
2. 回到 `20_interpretation/` 检查设计机会是否成立。
3. 回到 `30_framework/` 补充定位、策略和约束。
4. 要求 Visual Agent 重写提示词，并逐条说明与 Framework 的对应关系。
5. 重新交给 Critic Agent 评审。

## 最小 gate checklist

进入视觉生成前，必须回答：

- 我们理解了哪个真实场地或真实人群问题？
- 哪些研究结论支持这个问题？
- 设计机会是什么？
- 设计定位和核心命题是什么？
- 空间策略、视觉策略和运营约束是什么？
- 这个视觉方向为什么不是通用风格套壳？
- Critic Agent 将依据什么标准评审它？
