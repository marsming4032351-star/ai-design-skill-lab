一次任务怎么流动 从资料到复用

它关心的不是“生成一张图”

真正重要的是：这次为什么这样做，下次能不能少从零开始。

1
收集 brief / 文章 / 截图先进入 staging 或 references，不急着混进主项目。
2
生成 Plan / Prompt / 方向用已有 Pattern 和 Rule 帮助判断，不靠临场发挥。
3
评价和复审Critic、Review、测试、尺寸检查，让结果能被复查。
4
输出和归档HTML、PNG、post-copy、README，一起成为可追踪交付物。
5
下次复用成功经验变成 Pattern / Rule / 文档，而不是散落在聊天记录里。
这就是 Design Data Factory 的核心：让经验可以回流。 04 / 05
为什么值得做 给小白的总结

它把 AI 设计从“碰运气”变成“可复盘”

模型可以生成结果，但仓库负责保存方法、边界、证据和下一次复用的入口。

不只存图 还存来源、Prompt、规则、判断和导出方式。
不只跑脚本 每个 CLI 都对应一个可检查的生产阶段。
不只做 demo Harness 让任务生命周期、review 和 retry 可以被验证。
不只发内容 Output Skill 让图文包、deck 和发布文案成为资产。
你可以把它理解成：
资料库 + 规则库 + 执行流水线 + 输出规范 = 一个 AI 时代的设计工作台。
读仓库先看 MAP.md；做输出先看 OUTPUT_SKILL_SPEC.md。 05 / 05
