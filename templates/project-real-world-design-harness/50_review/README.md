# 50_review：评审阶段

这里存放 Critic Agent 和人工评审结果。

## 应该放什么

- 方案是否回应场地、历史、人群、甲方目标和运营落地的评审表。
- 美颜化设计风险检查。
- 通过 / 修改 / 拒绝判断。
- 修改建议和证据缺口。

## 不能做什么

- 不要只评价好不好看。
- 不要忽略运营、预算、维护和落地限制。
- 不要让未通过评审的输出进入 archive。

## 阶段产出

- `critic-review.md`
- `revision-notes.md`
- `approval-record.md`
- `beautification-risk-check.md`
