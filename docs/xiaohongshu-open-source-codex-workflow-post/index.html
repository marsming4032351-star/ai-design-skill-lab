Input 这个 GitHub 项目适合做什么？ Read README / AGENTS / index / examples Ask 它的机制、边界、可复用点是什么？ Output 先给结论，再决定是否克隆

Step 02

适合参考，才克隆到 references

外部项目不要直接混进成品目录。先放在参考区，保持边界清楚。

01

clone 到 references/

把它当外部参考库，而不是当前项目的一部分。

02

保留来源说明

记录 GitHub URL、用途、适合场景和使用限制。

03

必要时写入 .gitignore

完整第三方仓库通常不直接提交，只提交自己的评估文档。

Reference, not product03 / 05
Step 03

把项目读成自己的评估文档

README

它解决什么问题？作者希望用户怎样开始？

AGENTS

它希望 AI Agent 如何选择、复制和改造模板？

INDEX

是否有结构化索引？能不能机器读取和筛选？

WORKFLOW

它应该接入 PPT、网页，还是小红书卡片？

Step 04

最后看 git status，只提交该提交的东西

✓
提交自己的评估文档例如模板机制、接入建议、适用场景。
✓
提交真正的成品文件例如小红书的 HTML 源文件、发布文案和 PNG。
✓
忽略完整外部仓库第三方 checkout 留在本地 reference，避免撑大 Git 历史。
✓
不要顺手 git add .临时文件、缓存、授权输出、本机路径都要先排除。
