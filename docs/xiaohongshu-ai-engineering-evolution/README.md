# AI 工程三阶段演化图文包

主题：Prompt Engineering -> Context Engineering -> Harness Engineering。

## 文件

- `index.html`：可编辑的 5 页小红书图文源文件。
- `cards/`：导出的 1080 x 1350 发布图。
- `post-copy.md`：小红书标题、正文、评论引导和标签。

## 页数结构

1. 总览：AI 工程三阶段演化。
2. Prompt Engineering：研究怎么问问题。
3. Context Engineering：研究怎么给信息。
4. Harness Engineering：研究怎么驾驭 Agent。
5. 判断：不是纯噱头，也不是终局。

## 参考语境

- OpenAI 关于 Codex 与 Harness Engineering 的讨论：[Harness engineering: leveraging Codex in an agent-first world](https://openai.com/ms-BN/index/harness-engineering/)。
- Anthropic 关于 long-running agents / harness design 的讨论：[Harness design for long-running application development](https://www.anthropic.com/engineering/harness-design-long-running-apps)。
- Martin Fowler 网站对 Harness Engineering 的讨论：[Harness Engineering - first thoughts](https://martinfowler.com/articles/exploring-gen-ai/harness-engineering-memo.html)。
- LangChain 关于 agent harness 架构的讨论：[The Anatomy of an Agent Harness](https://www.langchain.com/blog/the-anatomy-of-an-agent-harness)。

> 说明：Harness Engineering 目前不是严格学术定义，图文采用工程圈较常见的解释：`Agent = Model + Harness`，即模型之外支撑 Agent 稳定工作的工具、权限、上下文、评估、反馈和运行环境。
