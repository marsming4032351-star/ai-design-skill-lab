# 小红书发布文案

## 标题

LLM Wiki 和 GBrain，到底有什么用？

## 正文

如果你还把 LLM 当聊天框，会漏掉下一波关键趋势：

AI 不只要会回答，还要会积累知识和记忆。

我会这样理解：

**LLM Wiki** 像一本会长大的知识库。它把项目资料、会议结论、设计规则、个人方法论沉淀成 Markdown 页面，可以检索、引用、复盘，也很适合接到 Obsidian 这样的笔记系统里。

**GBrain** 更像 Agent 的长期记忆层。它负责记住你的偏好、上下文、任务状态和工作方式，让 AI 不再每次都从零开始。

两者合起来，AI 才会从一次性问答工具，变成长期协作者。

我最推荐的普通人用法：

1. 用 Obsidian 管项目、灵感、会议、案例，把笔记写成结构化 Markdown。
2. 用 LLM Wiki 整理知识，让 AI 能检索和引用你的资料。
3. 用 GBrain 记住偏好、流程和长期上下文，让 Agent 更懂你。

一句话总结：

**Obsidian 是你的笔记库，LLM Wiki 是 AI 可读的知识层，GBrain 是 AI 记住你的方式。**

这套东西很适合设计策划、内容创作、研发协作、个人知识管理。
