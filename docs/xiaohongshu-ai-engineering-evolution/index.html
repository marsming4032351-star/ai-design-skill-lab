AI ENGINEERING EVOLUTION

AI 工程正在经历 三阶段演化

从“怎么问模型”，到“怎么喂信息”，再到“怎么给 Agent 搭一套可控系统”。Harness Engineering 火起来，核心就在这个变化里。

01
Prompt Engineering把话说清楚

优化提示词，让模型更准确理解需求。

02
Context Engineering把信息给对

管理上下文，把正确资料在正确时机交给模型。

03
Harness Engineering把系统搭稳

围绕模型设计工具、权限、反馈、评估和执行环境。

Prompt -> Context -> Harness 01 / 05
STAGE 01

Prompt Engineering：研究怎么问问题

它关注的是“发给模型的那句话”。早期模型不够稳，提示词写法会明显影响结果。

典型动作

补充角色、目标、约束、格式、示例，让模型减少误解。

粗糙问题：帮猫起名
更好问题：给一只橘色、活泼、爱玩的小猫起两个字的名字
✓
优点

上手快，立刻能改善回答质量。

!
局限

只改“怎么说”，解决不了复杂任务里的资料、工具、验证和执行问题。

关键词：提示词 / 约束 / 输出格式 02 / 05
STAGE 02

Context Engineering：研究怎么给信息

Context 不只是 prompt，还包括对话历史、检索资料、工具列表、规则、记忆和环境信息。

压缩

对长对话做总结，避免上下文被历史记录撑爆。

检索

需要时从文档、代码库、知识库里动态取资料。

披露

不一次性塞满信息，用到哪块再给哪块。

核心不是“给更多”，而是“在正确时机给正确内容”。
Prompt 是 Context 的一部分 03 / 05
STAGE 03

Harness Engineering：研究怎么驾驭 Agent

Harness 可以理解为模型外面的“控制系统”：让模型有工具可用、有规则可守、有反馈可改、有环境可跑。

Large
Model
工具管理

文件、浏览器、日志、CI、数据库、部署工具。

权限与边界

哪些能做、哪些要确认、哪些必须隔离。

验证反馈

测试、lint、截图、指标、评估器、回滚。

任务编排

规划、拆解、执行、审查、长期运行。

一个常见说法：Agent = Model + Harness 04 / 05
SO WHAT

它是突破，还是炒概念？

更准确的判断：Harness Engineering 不是全新技术发明，但它是当下很重要的系统工程框架。

1
不是凭空冒出来

测试框架、评估框架、工具链、CI、日志、权限，这些工程手段早就存在。

2
新在组织方式

它把零散工程能力组织成一套“让 Agent 稳定干活”的系统。

3
不是终局

模型越强，一部分 harness 会被模型能力吸收；但新任务也会需要新的 harness。

结论：Prompt 让 AI 听懂你，Context 让 AI 知道背景，Harness 让 AI 真正能干活。
