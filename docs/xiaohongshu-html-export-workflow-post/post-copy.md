# 小红书发布文案：如何把 HTML 导出小红书图片

## 标题

别再让 AI 直接生成满屏中文了

## 正文

今天这篇讲一个很实用的工作流：

**怎么把 HTML 导出成小红书 4:5 图片。**

我现在做教程型图文，基本不让 AI 直接生成带大量中文的图片。

原因不是 AI 不会画，而是：

- 中文容易糊；
- 长标题容易错；
- 改一个字要重来；
- 每张图尺寸不稳定；
- 很难把方法沉淀成模板。

更稳的做法是：

**HTML/CSS 负责排版，Playwright / 浏览器截图负责导出 PNG。**

一张小红书图，就写成一个独立的 HTML section：

```html
<section class="card" id="card-01">
  <h1>封面标题</h1>
  <p>这张图只讲一个观点</p>
</section>
```

然后把 `.card` 固定成：

```css
width: 1080px;
height: 1350px;
```

最后用浏览器按 `#card-01`、`#card-02` 这样的 id 逐张截图。

这样做的好处是：

1. 中文清晰；
2. 内容可改；
3. 尺寸稳定；
4. 可以复用成模板；
5. 适合 Codex 自动检查和导出。

如果你在做 AI 设计策划、品牌提案、小红书教程、工具拆解，这套方法比“直接让 AI 生成最终图”稳定很多。
