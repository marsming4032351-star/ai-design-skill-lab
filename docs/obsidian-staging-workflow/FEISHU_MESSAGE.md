# Design Data Factory 资料收集入口使用指南

Obsidian Staging 资料收集功能已经可以正式使用。

以后看到值得保留的微信文章、小红书案例、GitHub 项目、御炉品牌灵感，不用先想清楚归档到哪里，先用一个命令收进系统：

```bash
./scripts/capture.sh wechat "文章标题" "文章内容或链接"
```

生成的资料会进入：

```text
00_Inbox_Staging/
```

它会自动生成一份 Obsidian Markdown，记录标题、来源、原始内容、自动分类和后续处理建议。

常见用法：

```bash
./scripts/capture.sh wechat "北京老字号案例" "这是一篇关于北京老字号升级的微信文章"
```

也可以收集：

- 微信文章：复制链接、正文或摘要；
- 小红书案例：记录选题、封面、图文结构、评论反馈；
- GitHub 项目：直接放仓库链接；
- 御炉品牌灵感：记录北京、老字号、烤鸭、非遗、门头、菜单、空间体验等观察。

如果内容里出现御炉、烤鸭、北京、非遗、老字号等关键词，系统会自动识别为 `project: yulu`，方便后续整理到御炉品牌资产里。

现在这一步的定位是“先收进 Inbox，不让资料散掉”。下一步会继续做 `staging_to_asset`，把 `00_Inbox_Staging` 里的资料按项目、类型和价值归档成正式资产。
