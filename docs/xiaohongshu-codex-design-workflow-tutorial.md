# 小红书图文教程：用 Codex 做设计策划工作流

主题：用 Codex 把一个模糊设计需求，推进成可汇报的策划案、PPT 逻辑、视觉样张和 PDF。

适合账号定位：

- AI 设计工作流
- 品牌策划方法
- 设计提案效率
- 独立设计师 / 策划人 / 品牌顾问

建议发布形式：

- 小红书 10 张图文卡片
- 封面 1 张 + 教程步骤 8 张 + 总结 1 张
- 比例：3:4 或 4:5
- 风格：干净、专业、轻技术感、设计咨询公司感

## 第 1 张：为什么设计策划适合用 Codex

### 画面文案

标题：

> 设计策划不是只靠灵感

正文：

> 真正耗时间的是：
>
> - 读资料
> - 拆逻辑
> - 找定位
> - 重组 PPT
> - 检查交付物
>
> Codex 适合处理这些“流程型脑力劳动”。

### 讲解重点

设计策划表面看是创意，但实际包含大量信息处理、结构判断和反复验证。Codex 的价值不是替代审美，而是把杂乱输入变成可执行工作流。


## 第 2 张：完整工作流总览

### 画面文案

标题：

> 我的 Codex 设计策划流程

流程：

```text
01 输入资料
02 提取信息
03 头脑风暴
04 搭建策略
05 设计 PPT 逻辑
06 生成视觉样张
07 导出 PDF
08 复盘沉淀
```

底部小字：

> 重点不是“一次生成”，而是让每一步都有证据。

### 讲解重点

这张是全篇总览。要强调：Codex 工作流不是一个 prompt 解决全部，而是“输入、分析、产出、验证、沉淀”的链路。


## 第 3 张：第一步，把资料喂进去

### 画面文案

标题：

> Step 1：先让 Codex 读懂资料

可输入：

> - PPT / PDF / Word
> - 品牌资料
> - 图片资产
> - 竞品信息
> - 老版本方案

提示词示例：

```text
请先读取这两份 PPT，提取品牌定位、视觉资产、产品结构、空间叙事和可复用内容。
```

### 讲解重点

设计策划不要一上来让 AI “给我一个方案”。先让 Codex 读资料，提取结构，再判断哪些内容能保留、哪些需要重写。

### Lovart 提示词

```text
小红书图文卡片，主题 Step 1 资料输入。画面展示多个文件卡片：PPT、PDF、Word、图片、竞品资料，被汇入一个 Codex 工作台。风格现代、专业、清爽，米白背景，深灰线框，少量蓝色高亮。比例4:5。
```

## 第 4 张：第二步，做“诊断”而不是直接创作

### 画面文案

标题：

> Step 2：先诊断，再创作

让 Codex 判断：

> - 哪份方案逻辑更强？
> - 哪份方案视觉资产更好？
> - 哪些内容重复？
> - 哪些表达不够高级？
> - 哪些证据不足？

关键句：

> 不要简单合并，要判断主次。

### 讲解重点

很多设计提案失败，不是因为内容少，而是因为内容没有主线。Codex 可以帮你把多个版本拆成“策略骨架”和“视觉资产”。

### Lovart 提示词

```text
小红书教程卡片，主题“先诊断，再创作”。画面为两份方案文档被拆解成策略骨架和视觉资产两部分，中间有分析面板和勾选标记。风格像品牌咨询公司的工作台，米白底，深灰字，金色强调线，比例4:5。
```

## 第 5 张：第三步，头脑风暴定位

### 画面文案

标题：

> Step 3：把定位变成一句能记住的话

方法：

> 1. 找市场空位
> 2. 找品牌资产
> 3. 找用户理由
> 4. 找一句主张

提示词示例：

```text
请基于资料，给出 3 个品牌定位方向，并说明每个方向的优缺点和推荐理由。
```

### 讲解重点

Codex 适合做“多方案比较”。不要只让它给一个答案，而是要求它给 2-3 个方向、利弊、推荐方案。

### Lovart 提示词

```text
小红书图文卡片，主题“把定位变成一句能记住的话”。画面是品牌定位漏斗，从市场空位、品牌资产、用户理由，收束成一句品牌主张。高级策略咨询风，米白背景，黑色文字，金色重点，比例4:5。
```

## 第 6 张：第四步，搭 PPT 内容逻辑

### 画面文案

标题：

> Step 4：PPT 先搭逻辑，不急着排版

一套成立的提案逻辑：

```text
市场机会
↓
竞争卡位
↓
战略定位
↓
文化/产品证据
↓
体验系统
↓
视觉系统
↓
落地路径
```

关键句：

> 每一页都要回答一个问题。

### 讲解重点

正式做 PPT 前，先让 Codex 输出页级结构。每页明确：标题、核心观点、内容来源、版式建议。这样后面排版不会变成“漂亮但无逻辑”。

### Lovart 提示词

```text
小红书技术图文教程，主题“PPT先搭逻辑，不急着排版”。画面为一套PPT页级结构流程图，从市场机会到落地路径，每一步是一张简洁卡片。风格高级、专业、适合设计策划教程，比例4:5，米白背景，深灰字，金色箭头。
```

## 第 7 张：第五步，生成视觉样张

### 画面文案

标题：

> Step 5：先做 Look & Feel 样张

不用一开始做完整 PPT。

先做 5-8 个关键页型：

> - 封面
> - 市场判断
> - 战略原点
> - 产品锚点
> - 空间气质
> - 视觉系统
> - 落地路径

### 讲解重点

视觉样张的作用是定调。先确认风格，再扩成完整页数。这样效率比直接做完整 PPT 高。

### Lovart 提示词

```text
小红书图文卡片，主题“先做 Look & Feel 样张”。画面展示一组迷你PPT页面预览，包含封面、市场判断、战略原点、产品锚点、视觉系统等。风格像一线品牌设计公司提案，米白、黑、金配色，大留白，强网格，比例4:5。
```

## 第 8 张：第六步，把样张扩成完整提案

### 画面文案

标题：

> Step 6：从样张扩成完整提案

扩展规则：

> - 补目录
> - 补竞争卡位
> - 补客群
> - 补品牌语气
> - 补产品结构
> - 补空间动线
> - 补应用系统
> - 补落地模型

关键句：

> 页数不是越多越好，逻辑闭环才重要。

### 讲解重点

完整提案不是堆页面，而是补齐论证链。每页要么推进论证，要么提供证据，要么解释落地。

### Lovart 提示词

```text
小红书教程卡片，主题“从样张扩成完整提案”。画面展示一套PPT从7页扩展到26页的过程，左侧是少量样张，右侧是完整提案结构目录。风格清爽专业，信息图排版，米白底，深灰字，金色编号，比例4:5。
```

## 第 9 张：第七步，验证与导出

### 画面文案

标题：

> Step 7：别急着交付，先验证

我会检查：

> - 页数是否完整
> - 图片是否加载
> - 页面是否破版
> - 逻辑文档是否对应
> - PDF 是否能正常打开

提示词示例：

```text
请用浏览器打开预览文件，截图检查页面是否破版，并导出 26 页 PDF。
```

### 讲解重点

Codex 的强项是可以把“产出”和“验证”串起来。不是只生成内容，而是检查它能不能交付。

### Lovart 提示词

```text

```

## 第 10 张：总结页

### 画面文案

标题：

> Codex 做设计策划的关键

不是：

> 一个 prompt 直接出完

而是：

> 让 AI 参与完整工作流

核心公式：

```text

+ 结构诊断
+ 策略推演
+ 视觉样张
+ 交付验证

```

结尾：

> 收藏这套流程，下次做提案可以直接套。

## 小红书正文

```text
最近我用 Codex 跑了一套完整的设计策划工作流。

不是让 AI “帮我写个方案”这么简单，而是把它当成一个可以协作的设计策划助理：

1. 先读取 PPT / PDF / 图片资料
2. 提取品牌定位、视觉资产、产品结构
3. 判断哪份方案适合做策略骨架
4. 头脑风暴 2-3 个定位方向
5. 搭建 PPT 的页级逻辑
6. 生成 Look & Feel 视觉样张
7. 扩展成完整提案
8. 最后导出 PDF 并检查交付

我最大的感受是：

AI 不一定替你做最终审美判断，但它非常适合帮你把“混乱的设计过程”整理成一套可复用工作流。

尤其是品牌策划、空间提案、餐饮品牌定位、PPT 重构这类项目，Codex 的价值不只是写文案，而是：

- 读资料
- 拆逻辑
- 比较方案
- 生成结构
- 检查文件
- 沉淀方法

下次我可以继续整理：
“如何用 Codex 把两份 PPT 融合成一套高端设计公司风格提案”
```

## 可复用 Codex Prompt 模板

### 资料诊断 Prompt

```text
请先阅读这些资料，不要急着给方案。

请按以下结构输出：
1. 每份资料的核心内容
2. 可复用的品牌资产
3. 策略逻辑强的部分
4. 视觉资产强的部分
5. 重复、矛盾或证据不足的地方
6. 推荐的融合方向
```

### 品牌定位 Prompt

```text
基于以上资料，请给出 3 个品牌定位方向。

每个方向需要包含：
1. 一句话定位
2. 目标用户为什么会来
3. 品牌资产如何支撑
4. 产品/空间/视觉如何落地
5. 优点
6. 风险
7. 推荐程度
```

### PPT 逻辑 Prompt

```text
请把这个品牌策划案整理成一套正式汇报 PPT 结构。

要求：
1. 控制在 24-30 页
2. 每页只表达一个核心观点
3. 每页包含：页标题、核心观点、内容要点、视觉建议
4. 逻辑必须从市场机会推到品牌定位，再到产品、空间、视觉和落地路径
5. 不要堆砌空泛口号
```

### 视觉样张 Prompt

```text
请为这套 PPT 设计 6-8 个关键页型的 Look & Feel 样张。

页型包括：
1. 封面
2. 市场判断
3. 战略原点
4. 产品锚点
5. 空间气质
6. 视觉系统
7. 业态路径

风格要求：
- 高端设计咨询公司感
- 大留白
- 强网格
- 少色彩
- 图片和文字都服务一个核心判断
```

## 制作图片时的统一视觉规范

建议整组图保持一致：

- 画幅：4:5
- 背景：米白 / 浅灰
- 主字：深灰或黑色
- 强调色：金色 / 蓝色
- 字体感觉：现代黑体 + 少量衬线标题
- 页面元素：流程线、卡片、编号、文件图标、浏览器窗口、PPT 缩略图
- 避免：过度赛博、复杂 3D、花哨渐变、满屏小字

## 一句话总结

> Codex 做设计策划，最重要的不是“生成一个答案”，而是把资料、判断、结构、视觉和交付串成一条可复用工作流。
