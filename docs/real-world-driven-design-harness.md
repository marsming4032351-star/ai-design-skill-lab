# Real-World Driven Design Harness：现实驱动的设计编排系统

本文定义一套面向空间、品牌、餐饮门店、展馆等真实项目的设计编排方法。它的核心判断是：AI 不应该一开始就生成效果图。优秀设计的前 40%-50% 应该用于场地调研、历史研究、人群理解、甲方动机分析、情绪判断和设计机会提炼。

在 AI 时代，人类设计师的核心价值不是更快地画图，而是作为现实物理世界的感应器，捕捉现场的光、声音、尺度、人群、历史、情绪和约束。智能体的价值，是把这些现实输入转化为结构化研究、设计机会、设计框架、图像提示词、评审标准和可复用 Pattern。

## 核心理念

Real-World Driven Design Harness 的基本原则是：

```text
现实输入先于图像生成
研究解释先于视觉风格
设计框架先于效果图
评审标准先于审美判断
归档复用先于项目结束
```

这套系统把设计任务拆成一条可控链路：

```text
Human Sensor
  -> Research Agent
  -> Interpretation Agent
  -> Opportunity Agent
  -> Framework Agent
  -> Visual Agent
  -> Critic Agent
  -> Archive Agent
```

Harness 的作用不是替设计师做决定，而是控制每个阶段的输入、输出和准入条件，防止 AI 在没有理解现实的情况下直接进入“好看图片生产”。

## 为什么不能一上来画图

一上来画图的问题，不是效率太高，而是判断太少。

如果没有真实调研，AI 会默认从训练集里抽取“看起来像某类项目”的表面风格。空间设计会变成氛围图拼贴，品牌设计会变成 logo 风格选择，餐饮门店会变成网红装修，展馆设计会变成科技感背景板。这类结果可能很快、很漂亮，但很难回答：

- 场地真实问题是什么？
- 谁会使用这里？
- 甲方真正想解决什么商业或组织问题？
- 本地历史、街区气质、建筑尺度、运营限制是否被回应？
- 这个方案为什么只能发生在这里，而不是任何地方？

设计的前半段应该用来建立判断。效果图应该是判断的可视化结果，而不是判断的替代品。

## 人作为现实传感器的角色

Human Sensor 指人类设计师、项目负责人、调研者或现场协作者。他们的任务是把 AI 无法直接感知的现实世界带回系统。

人需要采集的不是“素材”，而是带判断的问题现场：

- 光线：自然光方向、明暗变化、夜间照明、反光和阴影。
- 声音：噪声、回声、街道声、设备声、人群活动声。
- 尺度：街道宽度、门头比例、室内层高、动线距离、人与物的相对尺度。
- 人群：谁经过、谁停留、谁消费、谁管理、谁被排除。
- 历史：地方记忆、建筑沿革、行业故事、品牌过往、社区关系。
- 情绪：紧张、松弛、热闹、冷清、庄重、亲密、陌生、可进入感。
- 约束：预算、消防、动线、设备、维护、出品速度、翻台、管理半径。
- 甲方动机：想增长、想转型、想招商、想提高客单、想建立文化势能，还是想解决运营混乱。

Human Sensor 的输出不是设计方案，而是可被智能体继续研究和解释的现实输入包。

## AI 作为执行器的角色

AI 在这套系统里不是“美图机器”，而是研究、解释、推演、评审和归档执行器。

不同智能体承担不同职责：

- Research Agent：整理事实、补充背景、识别资料缺口，不给设计方案。
- Interpretation Agent：把事实转译为矛盾、情绪、动机和设计机会。
- Opportunity Agent：提炼可被设计解决的问题，并区分真机会与伪机会。
- Framework Agent：形成设计定位、核心命题、关键词、空间策略、视觉策略和运营约束。
- Visual Agent：基于框架生成图像提示词，不脱离框架追求表面美感。
- Critic Agent：检查方案是否回应现实输入、历史、人群、甲方目标和运营落地。
- Archive Agent：把经过验证的判断、框架、提示词和评审结果归档成可复用 Pattern。

## Harness 如何避免 AI 沦为“美颜软件”

Harness 通过阶段门控制流程：

1. 没有 Human Sensor 输入，不允许进入 Research。
2. 没有 Research 输出，不允许进入 Interpretation。
3. 没有 Interpretation 和 Opportunity，不允许进入 Framework。
4. 没有通过检查的 Framework，不允许进入 Visual Agent。
5. Visual Agent 的提示词必须引用框架要素。
6. Critic Agent 不只评价好不好看，必须评价是否回应现实。
7. Archive Agent 只归档经过评审的经验，不归档一次性漂亮图。

这套控制把“图像生成”从起点改成中后段执行环节。AI 仍然可以生成图，但图必须服务于已经形成的设计判断。

## 完整工作流

### 1. Human Sensor

**输入**

- 现场照片、视频、手绘草图、尺寸记录、动线观察。
- 访谈记录、甲方口述、运营约束、预算和时间条件。
- 街区、历史、竞品、客群、社媒评论和现场情绪记录。

**输出**

- `00_input/field-notes.md`
- `00_input/site-photos/`
- `00_input/client-brief.md`
- `00_input/constraints.md`

**检查标准**

- 是否记录了时间、地点、观察者和采集方式。
- 是否同时包含事实和感受，但没有直接跳到方案。
- 是否标注了不确定信息和待补调研。

### 2. Research Agent

**输入**

- Human Sensor 输入包。
- 可公开检索的历史、行业、地方、品牌、竞品资料。

**输出**

- 场地事实表。
- 历史和语境研究。
- 人群与使用场景摘要。
- 资料缺口清单。

**检查标准**

- 只研究，不提出设计方案。
- 每个结论必须能追溯到输入、来源或明确假设。
- 区分事实、推断和问题。

### 3. Interpretation Agent

**输入**

- Research Agent 输出。
- Human Sensor 的情绪、矛盾和现场观察。

**输出**

- 关键矛盾。
- 情绪判断。
- 甲方真实动机解释。
- 初步设计机会。

**检查标准**

- 是否把资料转化为设计机会，而不是复述资料。
- 是否解释“为什么这个问题值得设计介入”。
- 是否区分用户机会、空间机会、品牌机会和运营机会。

### 4. Opportunity Agent

**输入**

- Interpretation Agent 的机会列表。
- 项目目标、预算、时程、运营能力。

**输出**

- 机会优先级。
- 机会-风险矩阵。
- 必须解决的问题和暂不解决的问题。

**检查标准**

- 是否说明每个机会的现实依据。
- 是否识别伪机会，例如只适合出图、不适合落地的概念。
- 是否给出取舍理由。

### 5. Framework Agent

**输入**

- 研究、解释和机会输出。
- 甲方目标、空间/品牌/运营约束。

**输出**

- 设计定位。
- 核心命题。
- 关键词。
- 空间策略。
- 视觉策略。
- 运营约束。
- 可进入 Visual Agent 的准入说明。

**检查标准**

- 是否明确“这个项目为什么应该这样设计”。
- 是否能指导后续视觉提示词，而不是停留在抽象口号。
- 是否包含不能做什么。

### 6. Visual Agent

**输入**

- Framework Agent 输出。
- 必要的参考图、材质、空间、品牌或图文输出规格。

**输出**

- 图像提示词。
- HTML / deck / 图文包结构。
- 视觉探索方向。

**检查标准**

- 提示词必须引用设计定位、核心命题和策略。
- 不允许脱离框架追求高级感、网红感、科技感等表面风格。
- 输出应说明每个视觉选择回应了哪个现实输入。

### 7. Critic Agent

**输入**

- Visual Agent 输出。
- Framework、Interpretation、Research 和原始输入。

**输出**

- 评价表。
- 通过 / 修改 / 拒绝判断。
- 缺失证据和修改建议。

**检查标准**

- 是否回应场地、历史、人群、甲方目标和运营落地。
- 是否识别“美颜化设计”风险。
- 是否指出哪些视觉只是好看但无现实依据。

### 8. Archive Agent

**输入**

- 通过评审的研究、框架、提示词、输出和 Critic 记录。

**输出**

- 可复用 Pattern。
- 项目复盘。
- Prompt 变体。
- Rule / checklist 更新建议。

**检查标准**

- 只归档经过评审的经验。
- 说明适用场景和不适用场景。
- 保留来源、判断和验证记录。

## 阶段准入总表

| 阶段 | 必须输入 | 主要输出 | 禁止事项 |
|---|---|---|---|
| Human Sensor | 现场和甲方现实输入 | 原始输入包 | 直接定风格 |
| Research Agent | 输入包和资料来源 | 事实、背景、缺口 | 给设计方案 |
| Interpretation Agent | 研究结果 | 矛盾、情绪、机会 | 只复述资料 |
| Opportunity Agent | 机会和约束 | 优先级、取舍 | 把所有想法都保留 |
| Framework Agent | 研究、解释、机会 | 定位、命题、策略 | 只写空泛口号 |
| Visual Agent | 设计框架 | 视觉提示词和输出结构 | 脱离框架追求好看 |
| Critic Agent | 输出和完整链路 | 评价、修改建议 | 只看审美 |
| Archive Agent | 通过评审的材料 | Pattern 和复盘 | 归档未验证结果 |

## 应用到空间设计

空间设计中的现实输入包括建筑尺度、光线、动线、材质、周边街区、设备管线、人群停留方式和维护成本。

应用方式：

1. Human Sensor 先记录现场尺度、光线、动线和使用冲突。
2. Research Agent 研究建筑、街区、功能和竞品。
3. Interpretation Agent 提炼空间矛盾，例如“可进入但缺仪式感”“面积小但需要被记住”。
4. Framework Agent 输出空间策略，例如入口序列、停留节点、视线组织和材料边界。
5. Visual Agent 只在框架通过后生成空间提示词。
6. Critic Agent 检查图像是否回应真实尺度、动线和运营。

## 应用到品牌设计

品牌设计中的现实输入包括品牌历史、创始人动机、客群认知、竞争语境、渠道触点和组织能力。

应用方式：

- Research Agent 不只看同行视觉，还要整理品牌资产和用户语言。
- Interpretation Agent 把资料转化为品牌矛盾，例如“想显得高级，但用户需要亲近感”。
- Framework Agent 输出品牌定位、语气、视觉关键词、符号边界和应用触点。
- Visual Agent 生成 logo、海报、社媒、门头等提示词时必须引用框架。
- Critic Agent 检查视觉是否只是换皮，还是支撑真实品牌目标。

## 应用到餐饮门店

餐饮门店尤其需要现实驱动，因为门店设计同时受翻台、出餐、动线、客单、坪效、门头吸引力和社媒传播影响。

应用方式：

- Human Sensor 记录人流、门头可见性、邻里关系、排队位置、出餐动线和高峰噪声。
- Research Agent 研究菜系、商圈、竞品和用户评价。
- Interpretation Agent 提炼消费场景和情绪，比如“日常可进，但有一桌请客的体面”。
- Framework Agent 输出空间策略、品牌语气、菜单叙事和运营约束。
- Visual Agent 生成门头、菜单、空间、社媒图时必须回应出餐和经营模型。
- Critic Agent 检查是否只做了网红效果，而没有解决翻台、识别和复购。

## 应用到展馆设计

展馆设计需要处理叙事、动线、信息密度、观众节奏、技术可维护性和机构目标。

应用方式：

- Human Sensor 记录场馆尺度、观众路径、停留时间、噪声、互动设备条件。
- Research Agent 整理主题资料、历史线索、展品关系和观众画像。
- Interpretation Agent 提炼叙事矛盾，例如“资料很重，但观众只能停留 20 分钟”。
- Framework Agent 输出总叙事、章节结构、空间节奏、互动策略和视觉语气。
- Visual Agent 生成展项画面、导视、沉浸空间提示词。
- Critic Agent 检查视觉是否服务叙事和参观节奏，而不是堆科技感。

## 最小落地方式

一次真实项目可以从以下文件开始：

```text
templates/project-real-world-design-harness/
├── 00_input/
├── 10_research/
├── 20_interpretation/
├── 30_framework/
├── 40_output/
├── 50_review/
└── 60_archive/
```

配套使用：

- `prompts/research_agent.md`
- `prompts/interpretation_agent.md`
- `prompts/framework_agent.md`
- `prompts/visual_prompt_agent.md`
- `prompts/critic_agent.md`
- `rules/no-drawing-before-framework.md`

这套方法的目标不是让 AI 少做图，而是让 AI 画出来的每一张图都带着现实依据、设计判断和可复用价值。
