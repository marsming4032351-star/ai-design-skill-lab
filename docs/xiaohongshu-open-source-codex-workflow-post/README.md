# 小红书发布包：从刷到开源项目到 Codex 分析沉淀

这是一个面向小红书发布的知识经验分享图文包。

主题：看到视频或图文里提到的 GitHub 开源项目后，如何用 Codex 完成分析、克隆、评估、沉淀和 Git 提交边界管理。

## 文件说明

- `index.html`：可编辑的 5 页小红书图文源文件。
- `post-copy.md`：发布标题、正文、评论区引导和标签。
- `assets/gpt-image-open-source-codex-cover.png`：GPT Image 生成的封面氛围背景。
- `cards/card-01.png` 到 `cards/card-05.png`：导出的 1080 x 1350 发布图。

## 页数结构

1. 封面：刷到一个开源项目，别只收藏。
2. 第一步：先让 Codex 读懂项目。
3. 第二步：克隆到 references，而不是直接混进成品。
4. 第三步：把 README / AGENTS / index 变成自己的评估文档。
5. 第四步：提交前看 git status，只提交该提交的东西。

- 数量：5 张
- 尺寸：1080 x 1350 px
- 比例：4:5
- 格式：PNG
- 风格：AI developer workspace / GitHub research / Codex workflow，暗色技术氛围 + 稳定中文排版
